DECOMPOSITION RULES
────────────────────────────
1. Each output must be a single line:
   Robot:Skill > Robot:Skill > Robot:Skill …

   a. If the request involves *finding, scanning, or looking for* → use Aerial_Scan.
   b. If *identifying or verifying objects/people* → add Object_Recognition.
   c. If *recording or viewing remotely* → include Live_Video_Relay.
   d. If *speaking, warning, or playing audio* → use Audio_Announcement.
   e. If *picking up or delivering an item* → include Pickup_From_Merchant, Sidewalk_Navigation, Secure_Handoff (in this order).
   f. If *inside a building* → add Building_Access and/or Indoor_Navigation.
   g. If *elevator or multi-floor movement* → include Elevator_Operation.
   h. If *cleanup or safety verification* → use Environmental_Sensing, Minor_Repair, or QR_Tagging.
   i. If *collecting trash or recyclables* → add Waste_Collection.
   j. If *marking or signaling a spot* → use Location_Ping.
   k. If *responding quickly* → include Rapid_Dispatch.
   l. If *human interaction or confirmation* → include Video_Call_Setup or Staff_Alert.

3. Always order skills logically:
   - Perception → Action → Verification
   (e.g., Aerial_Scan before Object_Recognition before Environmental_Sensing).

4. Include at least two robots when possible.

5. Use no extra words or explanations. 
   Output only the ordered plan.
